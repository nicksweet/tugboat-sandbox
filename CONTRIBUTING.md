# CONTRIBUTING.md

## Purpose

This document provides guidelines for contributors to the Tugboat sandbox repository, specifically for running end-to-end (E2E) tests of the tugboat CLI.

## Prerequisites

Before you begin, ensure you have the following installed and configured:

- [Tugboat](https://github.com/yourorg/tugboat) (the CLI tool) installed and configured with access to a frontier LLM API (e.g., OpenAI, Anthropic).
- [GitHub CLI (gh)](https://cli.github.com/) installed and authenticated with your GitHub account.
- Access to the sandbox repository with appropriate permissions to create branches, issues, and pull requests.

## How to run tugboat plan (E2E)

Use `tugboat plan` to decompose a feature into tasks with frontier model assistance.

**Prerequisites**
- Tugboat installed and configured
- Access to a frontier LLM API (e.g., OpenAI, Anthropic)
- A GitHub repository with feature description

**Command syntax**
```bash
tugboat plan --brief "..." --project <id>
tugboat plan --brief-file path/to/brief.md [--project <id>] [--plan-only] [--yes]
```
Requires exactly one of `--brief` or `--brief-file`.

**Example usage**
```bash
# Plan a new feature called "feature/auth" using a brief string
tugboat plan --brief "Add user authentication with login/logout endpoints" --project 123

# Plan using a brief file, generating plan only without creating tracker issue
tugboat plan --brief-file ./docs/feature/auth.md --project 123 --plan-only
```

## How to run tugboat run-feature (E2E)

Use `tugboat run-feature` to execute the plan tracker issue, creating one scoped commit per task.

**Prerequisites**
- Tugboat installed and configured
- A plan tracker issue generated by `tugboat plan` (with priority label)
- Access to GitHub with permissions to create branches and commits

**Command syntax**
```bash
tugboat run-feature --project <id> --issue <N> [--dry-run] [--continue] [--yes]
```
Consumes a **plan tracker issue** (priority label only — not `agent-ready`).
Checks out the **Branch** from the tracker, loads tasks from **Spec root**, one scoped commit per task.

**Example usage**
```bash
# Execute the plan tracker issue #456 in project 123
tugboat run-feature --project 123 --issue 456

# Dry run to preview what would be done
tugboat run-feature --project 123 --issue 456 --dry-run

# Continue from a previous interruption
tugboat run-feature --project 123 --issue 456 --continue
```
